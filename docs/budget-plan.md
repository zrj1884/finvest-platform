### 方案 C：创始人独立 + AI 开发模式（极致省钱，推荐）

创始人本人即技术负责人，不雇佣任何全职员工，利用 Claude Code 独立完成全部开发。

#### 1.8 核心理念

- **零工资支出**：创始人自己干，不领工资，以时间换资金
- **一人 + AI 全栈**：Claude Code 充当后端、前端、AI、测试全部角色
- **能省则省**：所有付费项优先找免费替代，非必要不花钱
- **延后非必要支出**：合规、安全审计等推迟到产品验证后再投入
- **MVP 优先**：先做核心功能上线验证，再逐步完善

### 方案 B：极致降本部署（推荐）

#### 2.1 核心原则

1. **零 GPU**：所有大模型推理走 API，不自采/租用任何显卡
2. **优先免费**：开源自建 > 云厂商免费额度 > 低价方案
3. **按量付费**：初期用量小，避免任何包年/预留实例
4. **轻量架构**：初期用单机 Docker Compose 替代 K8s，降低运维复杂度
5. **国产优先**：同等能力下优先选择国内低价 API（DeepSeek 等）

#### 2.2 AI 推理成本（API 调用替代 GPU）

| AI 功能模块 | 推荐 API 方案 | 单价参考 | 预估月调用量 | 月费用（元） |
|-------------|-------------|---------|-------------|-------------|
| 金融 LLM（情感分析、投研报告、智能助手） | DeepSeek API（V3/R1） | 输入 ¥1/百万 token，输出 ¥2/百万 token（缓存命中 ¥0.1） | 5000 万 token | 500-1500 |
| 金融 LLM 备选 | 阿里通义千问（Qwen-Plus） | 输入 ¥0.8/百万 token，输出 ¥2/百万 token | 备用 | 按需 |
| 金融 LLM 备选 | 硅基流动 SiliconCloud（开源模型托管） | 部分模型免费，付费低至 ¥0.35/百万 token | 备用 | 按需 |
| 情感分析批量处理 | DeepSeek API（批量模式） | 半价优惠 | 新闻/舆情数据 | 200-500 |
| Agent 多轮协作（多 Agent 讨论） | DeepSeek R1（推理模型） | 输入 ¥4/百万 token（缓存 ¥1），输出 ¥16/百万 token | 1000 万 token | 500-2000 |
| 复杂推理备选 | Claude API（Sonnet 4.6） | $3/百万输入，$15/百万输出 | 仅关键场景 | 按需 |
| **月合计** | | | | **1200-4000 元** |
| **年合计** | | | | **1.4-4.8 万** |

> 对比：自建 GPU 方案年费 25-110 万 → API 方案年费 1.4-4.8 万，**节省 95% 以上**。

### 方案 B 的扩展路径

随用户量增长，可按需升级：

| 用户规模 | 架构调整 | 预估月成本 |
|---------|---------|-----------|
| < 100 人 | 单机 Docker Compose | 500-2000 元 |
| 100-1000 人 | 2-3 台云服务器 + 负载均衡 | 2000-5000 元 |
| 1000-10000 人 | 轻量 K8s（3-5 节点）+ 独立数据库 | 5000-15000 元 |
| > 10000 人 | 完整 K8s 集群，评估自建 GPU 是否比 API 更划算 | 按需评估 |

> 只有当 API 调用费用持续超过自建 GPU 成本时，才考虑自采显卡。按当前 DeepSeek 定价，预计日均 10 万次以上 LLM 调用才达到这个拐点。

## 四、AI 大模型灵活性分析

### 4.1 统一 LLM 网关架构（API 模式天然支持）

采用 API 调用模式后，模型切换变得极其简单，只需更改配置即可：

```text
应用层 → 统一 LLM 网关（路由 + 限流 + 计费追踪）
                ├── DeepSeek API（默认，性价比最高）
                ├── 通义千问 API（备选，国产生态）
                ├── 硅基流动 SiliconCloud（免费/低价开源模型）
                ├── Claude API（复杂推理场景）
                ├── OpenAI API（兼容性备选）
                └── 本地模型（未来自建时平滑切换）
```

### 4.2 推荐的模型分级策略

| 任务类型 | 推荐模型 | 原因 |
|---------|---------|------|
| 新闻情感分析（批量） | DeepSeek V3 | 价格最低，批量半价 |
| 投研报告生成 | DeepSeek V3 / Qwen-Plus | 中文金融语料表现好 |
| 多 Agent 推理决策 | DeepSeek R1 | 推理能力强，性价比高 |
| 复杂金融逻辑 / 关键决策 | Claude Sonnet 4.6 | 准确率最高，关键场景使用 |
| 用户对话助手 | DeepSeek V3 | 响应快、价格低 |
| 简单分类/摘要 | 硅基流动免费模型 | 零成本 |

### 4.3 切换零成本的保障措施

1. **统一 LLM 网关层**：所有 AI 调用通过统一接口，后端配置切换，业务代码零改动
2. **Prompt 模板管理**：按模型维护 prompt 变体，切换时自动适配格式
3. **成本追踪仪表盘**：实时监控各模型 token 消耗和费用，数据驱动优化选型
4. **自动降级机制**：主模型 API 不可用时，自动路由到备选模型
5. **A/B 测试**：新模型上线前，小流量验证效果后再全量切换

## 六、降本建议

1. **开发模式**：推荐方案 C（创始人+AI），首年总投入仅 3-10 万，以时间换资金
2. **AI 推理**：以 DeepSeek API 为主力（当前国内性价比最高），Claude/Qwen 为备选，通过统一网关随时切换
3. **ML 训练**：善用 Google Colab / Kaggle 免费 GPU 额度，仅重训练时按需租用 AutoDL
4. **数据源**：AKShare 免费覆盖绝大部分需求，付费数据源等确有需要时再接入
5. **基础设施**：初期单机 Docker Compose 起步，善用云厂商新用户优惠，按增长节奏扩展
6. **监控运维**：Prometheus + Grafana + Loki + Sentry 全套开源零成本
7. **持续关注**：AI API 价格战持续中（DeepSeek、通义、硅基流动等竞相降价），定期评估切换到更低价方案
8. **产品策略**：MVP 先行，核心功能验证后再投入非必要模块（移动端、完整安全审计等）
