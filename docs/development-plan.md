# FinVest Platform - 研发计划

## 总体规划

- **项目周期**: 约 12 个月（6个阶段）
- **团队建议**: 后端 3-4人、前端 2人、AI/算法 2-3人、数据工程 1-2人、测试 1人、产品 1人
- **开发方式**: 敏捷迭代，2周一个 Sprint

---

## 第一阶段 P0：基础设施与数据基座（第 1-8 周）

### S1.0 目标

搭建项目骨架、基础设施和数据采集管道，为后续所有模块提供数据支撑。

### Sprint 1-2（第 1-4 周）：项目初始化与基础设施

#### S1.1 项目工程化搭建

- [ ] 初始化 Git monorepo，配置 CI/CD（GitHub Actions）
- [ ] 搭建后端项目骨架（FastAPI + Poetry）
- [ ] 搭建前端项目骨架（Vue 3 + Vite + TypeScript）
- [ ] 配置 Docker Compose 本地开发环境
- [ ] 编写项目 README 和贡献指南

#### S1.2 数据库设计与部署

- [ ] 设计核心数据模型（用户、账户、持仓、交易、行情）
- [ ] 部署 PostgreSQL + TimescaleDB
- [ ] 部署 Redis 缓存层
- [ ] 编写数据库迁移脚本（Alembic）
- [ ] 设计时序数据分区策略（按日/月分区）

#### S1.3 消息队列与基础中间件

- [ ] 部署 Kafka 集群（行情数据流）
- [ ] 部署 RabbitMQ（任务调度）
- [ ] 实现基础消息生产/消费框架
- [ ] 配置 Prometheus + Grafana 监控

### Sprint 3-4（第 5-8 周）：数据采集管道

#### S1.4 行情数据接入

- [ ] 集成 AKShare 获取 A股实时/历史行情
- [ ] 集成 Yahoo Finance 获取美股行情
- [ ] 集成富途 OpenAPI / 长桥 SDK 获取港股行情
- [ ] 实现行情数据标准化（统一字段命名、时区处理）
- [ ] 行情数据写入 TimescaleDB，建立连续聚合视图
- [ ] 实现基金净值数据采集（天天基金/东方财富）
- [ ] 实现债券数据采集（中证估值、上清所）

#### S1.5 资讯与舆情采集

- [ ] 实现新闻爬虫框架（参考 BettaFish MindSpider）
- [ ] 接入财经新闻源（新浪财经、东方财富、Reuters）
- [ ] 接入社交舆情源（雪球、微博财经大V）
- [ ] 实现公司公告采集（巨潮资讯、港交所披露易）
- [ ] 宏观经济数据采集（央行、统计局、美联储 FRED）
- [ ] 数据清洗与去重管道
- [ ] 所有采集任务注册到调度系统（定时/实时）

#### S1.6 数据API层

- [ ] 实现内部数据查询 REST API
- [ ] 行情快照 / 历史K线 / 财务数据接口
- [ ] 数据缓存策略（Redis 热点数据）
- [ ] API 限流与鉴权

### P0 交付物

- 可运行的开发环境（Docker Compose 一键启动）
- 5个市场的行情数据持续入库

- 内部数据查询 API

## 第二阶段 P1：AI 预测引擎（第 9-18 周）

构建核心 AI 能力，包括走势预测模型、情感分析和投研报告生成。

### Sprint 5-6（第 9-12 周）：ML 预测模型

#### S2.1 特征工程管道

- [ ] 构建技术指标计算模块（MA/MACD/RSI/KDJ/布林带等）
- [ ] 构建基本面因子（PE/PB/ROE/营收增长率等）
- [ ] 构建情绪因子占位（后续接入 FinGPT）
- [ ] 实现特征存储（Feature Store）
- [ ] A股/美股/H股分别建立因子库

#### S2.2 股票预测模型（参考 Qlib）

- [ ] 搭建模型训练框架（PyTorch + MLflow 实验管理）
- [ ] 实现 LSTM 基线模型
- [ ] 实现 Transformer 模型
- [ ] 实现 TCN（时间卷积网络）模型
- [ ] 模型集成（Ensemble）策略
- [ ] 回测框架搭建（历史数据回放 + 收益/风险指标计算）
- [ ] A股/美股/H股分别训练与评估

#### S2.3 债券分析模型

- [ ] 集成 QuantLib 债券定价引擎
- [ ] 实现利率期限结构拟合（Nelson-Siegel / Svensson）
- [ ] 信用利差模型
- [ ] 久期/凸性计算与风险度量
- [ ] 可转债转股价值与转股溢价率模型

### Sprint 7-8（第 13-16 周）：金融 LLM 与情感分析

#### S2.4 金融 LLM 部署（参考 FinGPT）

- [ ] 选择基座模型（Qwen / Llama 3）
- [ ] 收集金融语料（财报、研报、新闻）
- [ ] LoRA 微调金融 LLM
- [ ] 部署模型推理服务（vLLM / TGI）
- [ ] 实现新闻情感分析管道（实时打分）
- [ ] 情绪因子回写至特征库

#### S2.5 投研报告 Agent（参考 FinRobot）

- [ ] 设计 Agent 工作流（数据收集 → 分析 → 报告生成）
- [ ] 实现个股分析报告模板
- [ ] 实现行业对比报告模板
- [ ] 实现债券评级分析报告模板
- [ ] 报告输出为 PDF / HTML / Markdown

### Sprint 9（第 17-18 周）：强化学习仓位管理

#### S2.6 DRL 交易策略（参考 FinRL）

- [ ] 搭建交易环境（Gym 风格模拟器）
- [ ] 实现 PPO 策略
- [ ] 实现 SAC 策略
- [ ] 多资产组合优化（股票+债券+基金混合）

## 第三阶段 P2：交易系统（第 19-28 周）

### S3.0 目标

实现多市场交易执行能力，从模拟盘到实盘。

#### S3.1 统一交易接口设计

- [ ] 定义统一交易协议（下单/撤单/查询/回调）
- [ ] 实现 Go 交易网关框架（高并发、低延迟）
- [ ] 实现订单状态机（待提交→已提交→部分成交→全部成交/已撤销）
- [ ] 实现订单持久化与恢复机制

#### S3.2 A股交易通道

- [ ] 对接券商交易API（如华泰 easytrader / 同花顺）
- [ ] 实现A股交易规则引擎（T+1、涨跌停、集合竞价）
- [ ] 可转债交易（T+0）

#### S3.3 美股交易通道

- [ ] 对接 Alpaca API（零佣金）
- [ ] 对接 Interactive Brokers TWS API（备选）
- [ ] 实现美股交易规则（T+0、盘前盘后、熔断机制）

#### S3.4 港股交易通道

- [ ] 对接富途 OpenD（港股通）
- [ ] 对接盈透 IB Gateway
- [ ] 实现港股交易规则（T+0交收T+2、买卖价差、最小交易单位）

#### S3.5 基金与债券通道

- [ ] 基金申购/赎回接口

- [ ] 可转债打新申购

### Sprint 12-13（第 23-26 周）：策略引擎与风控

#### S3.6 策略回测引擎

#### S3.7 模拟交易系统

## 技术风险与应对

| 风险 | 影响 | 应对方案 |
| --- | --- | --- |
| 券商API接入受限 | 交易功能不可用 | 优先对接开放度高的平台（Alpaca/富途），预留手动交易兜底 |
| AI模型预测准确率不足 | 用户信任度下降 | 设定保守策略为默认，强调风险提示，持续迭代模型 |
| 行情数据延迟 | 交易决策滞后 | 多数据源冗余，实时监控延迟指标 |
| 金融监管政策变化 | 业务合规风险 | 模块化设计便于按地区裁剪功能，提前咨询法律顾问 |
| 数据源不稳定 | 采集中断 | 多源备份，故障自动切换，数据质量监控 |
